# Структура базы данных ЮК «Фемида»

База данных спроектирована для поддержки публичного каталога, личного кабинета пользователя/юриста и админ. панели.

### 1. Пользователи
Хранит данные всех зарегистрированных лиц. Роли разделяются полем role.

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `INT PK AUTO_INCREMENT` | Уникальный идентификатор |
| `email` | `VARCHAR(255) UNIQUE` | Логин для входа |
| `password_hash` | `VARCHAR(255)` | Хешированный пароль |
| `first_name` | `VARCHAR(100)` | Имя |
| `last_name` | `VARCHAR(100)` | Фамилия |
| `phone` | `VARCHAR(20)` | Телефон |
| `role` | `ENUM('user','specialist','admin')` | Роль в системе |
| `created_at` | `TIMESTAMP` | Дата регистрации |
| `updated_at` | `TIMESTAMP` | Дата последнего изменения |

### 2. Категории услуг
Группировка юридических услуг для фильтрации в каталоге.
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `INT PK AUTO_INCREMENT` | Уникальный идентификатор |
| `name` | `VARCHAR(100)` | Название категории |
| `slug` | `VARCHAR(100) UNIQUE` | ЧПУ-идентификатор для URL |
| `created_at` | `TIMESTAMP` | Дата создания |

### 3. Юридические услуги
Основная контентная сущность. Отображается в `/services` и `/services/{id}`.
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `INT PK AUTO_INCREMENT` | Уникальный идентификатор |
| `category_id` | `INT FK → categories(id)` | Привязка к категории |
| `title` | `VARCHAR(255)` | Название услуги |
| `description` | `TEXT` | Описание, условия, состав |
| `price` | `DECIMAL(10,2)` | Стоимость (0.00 если договорная) |
| `is_active` | `TINYINT(1)` | Публикация (1/0) |
| `created_at` | `TIMESTAMP` | Дата добавления |
| `updated_at` | `TIMESTAMP` | Дата обновления |

### 4. Статусы заявок
Справочник состояний операционной сущности.
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `INT PK AUTO_INCREMENT` | Уникальный идентификатор |
| `name` | `VARCHAR(50)` | Человекочитаемое название |
| `slug` | `VARCHAR(50) UNIQUE` | Технический идентификатор |
| `description` | `VARCHAR(255)` | Пояснение для админки |

### 5. Заявки на консультацию
Операционная сущность. Связывает пользователя, услугу, статус и специалиста.
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `INT PK AUTO_INCREMENT` | Уникальный идентификатор |
| `user_id` | `INT FK → users(id)` | Кто оставил заявку |
| `service_id` | `INT FK → services(id)` | На какую услугу заявка |
| `specialist_id` | `INT FK → users(id) NULL` | Назначенный юрист (заполняется позже) |
| `status_id` | `INT FK → statuses(id)` | Текущий статус |
| `client_message` | `TEXT` | Суть вопроса от клиента |
| `specialist_response` | `TEXT NULL` | Ответ/консультация специалиста |
| `created_at` | `TIMESTAMP` | Дата создания |
| `updated_at` | `TIMESTAMP` | Дата изменения статуса |

## Связи между таблицами
categories 1 ─── * services

users 1 ─── * applications (как клиент)

users 1 ─── * applications (как специалист)

services 1 ─── * applications

statuses 1 ─── * applications
